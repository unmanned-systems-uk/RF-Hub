Software Beamforming — Steering in Code

How digital beamforming applies complex weights to sampled signals in software — from the steering vector to MVDR adaptive null steering.

📚 Unit 4: Advanced Systems ⚡ Prerequisites: Lesson 19, Python familiarity helpful 🛠 Equipment: RTL-SDR or HackRF (optional) ⏱ 40 minutes Advanced

CHAPTER 02

What Software Beamforming Actually Does

In Lesson 19, we applied phase shifts in hardware — lengths of coaxial cable, PIN diode switches. Digital beamforming (DBF) replaces this with arithmetic. Each array element has its own receiver that outputs a stream of complex (I/Q) samples. These sample streams are multiplied by complex weights and summed. That's the entire algorithm.

📐
The beamformer output:
y(t) = w₁* · x₁(t) + w₂* · x₂(t) + ... + wN* · xN(t)

where xₙ(t) is the complex sample from element n, wₙ is the complex weight for element n, and * denotes complex conjugate. In matrix notation: y = wᴴx

The key insight: a complex number has both amplitude and phase. Multiplying a signal by e^(jφ) shifts its phase by φ without changing its amplitude. By choosing the right complex weights, you implement any combination of phase shifts and amplitude tapers across the array — in a single line of code per time sample.

The power of DBF lies in what happens next: you can apply different weight vectors to the same raw data simultaneously, forming multiple beams in different directions without any additional hardware. The same received data streams serve all the beams.

CHAPTER 03

Complex Weights — Amplitude and Phase as a Single Number

A complex weight wₙ = Aₙ · e^(jφₙ) encodes two things simultaneously:

  • Aₙ — the amplitude weighting (tapering) applied to element n. Setting Aₙ < 1 suppresses that element's contribution, implementing amplitude tapering.
  • φₙ — the phase shift applied to element n. This is exactly the progressive phase shift from Lesson 17.

For a beam steered to angle θ with uniform amplitude (no tapering), the weight for element n is simply:

Steering Weight — Element n in a ULA
wₙ = e^(-j · n · kd · sin θ) = e^(-j · n · 2π · d · sin θ / λ)

where n = 0, 1, 2, ..., N−1 (element index)
This is the complex conjugate of the steering vector aₙ.

In Python, using NumPy, this is a one-liner:

# Steering weights for ULA, element indices 0..N-1 n = np.arange(N) w = np.exp(-1j * n * 2*np.pi * d_over_lambda * np.sin(theta_rad)) # Apply to received data matrix X (shape N x T): one row per element y = w.conj() @ X # shape: (T,) — beamformed output samples
CHAPTER 04

Steering Vector — Geometry Maps to Phase Differences

The steering vector a(θ) describes how a signal from direction θ appears across all elements of the array. For a Uniform Linear Array (ULA) with N elements, element spacing d, and wavelength λ:

Steering Vector — N-element ULA
a(θ) = [1, e^(j·kd·sinθ), e^(j·2kd·sinθ), ..., e^(j·(N-1)kd·sinθ)]ᵀ

where k = 2π/λ (wavenumber)

In Python:
a = np.exp(1j * n * 2*np.pi * d_over_lambda * np.sin(theta_rad))

The steering vector is the array's "fingerprint" of a signal arriving from θ. Element 0 is the phase reference (phase = 0). Each subsequent element accumulates more phase delay because the signal arrives later. The beamformer exploits this: the optimal weight vector to receive a signal from θ is the conjugate of the steering vector.

For non-ULA geometries (2D planar arrays, irregular arrays), the steering vector formula generalises by replacing the 1D path difference n·d·sinθ with the full 3D dot product between the element position vector and the direction-of-arrival unit vector.

CHAPTER 05

Delay-and-Sum Beamforming — The Simplest Beamformer

Delay-and-sum (DAS) is the most straightforward DBF algorithm. To receive from direction θ:

1

Compute the steering vector a(θ)

Calculate the expected inter-element phase progression for direction θ using the formula above.

2

Apply conjugate weights: w = a*(θ) / N

The conjugate phases "undo" the propagation delays, aligning the signals from direction θ before summation. Dividing by N normalises the output.

3

Compute output: y = wᴴ x

The inner product of weight vector and received sample vector. Signals from θ add coherently (gain = N). Signals from other directions partially cancel.

4

Repeat for each time sample

Apply the same weights to each column of the received data matrix. Output is a 1D time series representing the beamformed signal.

# Delay-and-sum beamformer in NumPy import numpy as np N = 4 # number of elements d_over_lam = 0.5 # spacing in wavelengths (λ/2) theta_deg = 30 # desired beam direction (degrees) theta = np.radians(theta_deg) n = np.arange(N) # Steering vector a = np.exp(1j * n * 2*np.pi * d_over_lam * np.sin(theta)) # Delay-and-sum weight vector (normalised) w_das = a.conj() / N # Beamform: X has shape (N, T) — N elements, T time samples # y = w_das.conj() @ X gives (T,) beamformed output

DAS is simple, robust, and computationally cheap. Its limitation is that the sidelobe level is fixed by the array aperture and window — it cannot adaptively place nulls on interferers.

CHAPTER 06

Limitations of Delay-and-Sum

LimitationWhy it occursConsequence
Fixed sidelobe level DAS weights are determined by geometry only, not by the interference environment A strong interferer in a sidelobe degrades SNR
No null steering DAS cannot place a null precisely on an interferer's direction Interferer rejection depends only on sidelobe level at that angle
Resolution limit Minimum resolvable angle = λ/(N·d) — the Rayleigh limit Two closely-spaced sources cannot be separated if their angular separation < λ/(N·d)
Signal mismatch sensitivity Steering vector errors (due to calibration errors or multipath) degrade gain Robust DAS (with steering vector uncertainty) is needed for real systems

These limitations motivate adaptive beamforming algorithms that use the actual received data statistics to optimise the weight vector — instead of relying on geometry alone.

CHAPTER 07

MVDR Beamforming — Minimum Variance Distortionless Response

MVDR (also called Capon beamforming, 1969) solves an optimisation problem: find the weight vector that minimises the total output power while keeping the response to signals from the desired direction exactly 1 (unity gain, no distortion). This forces nulls onto all interference sources automatically.

MVDR Weight Vector
w_MVDR = (R⁻¹ · a(θ)) / (aᴴ(θ) · R⁻¹ · a(θ))

where:
R = (1/T) · X · Xᴴ is the sample covariance matrix (N × N)
a(θ) is the steering vector for the desired direction
R⁻¹ is the matrix inverse of R

The covariance matrix R captures the statistical structure of the received signals — including which directions interferers come from. Inverting R effectively "whitens" the interference, and the subsequent projection onto a(θ) places nulls precisely where the interferers are, without being told where they are.

# MVDR beamformer in NumPy # X: (N, T) received data matrix # 1. Estimate covariance matrix R = (X @ X.conj().T) / X.shape[1] # 2. Regularise (diagonal loading for numerical stability) R_reg = R + 1e-6 * np.eye(N) # 3. Invert R_inv = np.linalg.inv(R_reg) # 4. Compute MVDR weights a = np.exp(1j * n * 2*np.pi * d_over_lam * np.sin(theta)) Rinv_a = R_inv @ a w_mvdr = Rinv_a / (a.conj() @ Rinv_a) # 5. Beamform y_mvdr = w_mvdr.conj() @ X
⚠️
MVDR requires enough data: The covariance matrix R must be estimated from data. A rule of thumb: use at least 2N–10N time samples to get a reliable estimate. With too few samples, R is poorly conditioned and its inverse is unreliable. Diagonal loading (adding a small constant to the diagonal) regularises the inversion.
CHAPTER 08

MUSIC — Finding Angles by Eigenvalue Decomposition

MUSIC (MUltiple SIgnal Classification, Schmidt 1986) is a direction-of-arrival (DOA) estimation algorithm rather than a beamformer. It finds the directions of incoming signals with very high resolution — potentially super-resolution, beyond the Rayleigh limit.

The key idea: the covariance matrix R can be decomposed into a signal subspace and a noise subspace. The steering vectors of the incoming signals lie in the signal subspace and are therefore orthogonal to the noise subspace.

MUSIC Pseudo-Spectrum
P_MUSIC(θ) = 1 / (aᴴ(θ) · Eₙ · Eₙᴴ · a(θ))

where Eₙ is the matrix of noise eigenvectors (columns of U from SVD,
corresponding to the N−K smallest eigenvalues for K sources).

P_MUSIC peaks sharply at the true DOA angles.

Delay-and-Sum

  • Simple — no matrix operations
  • Robust to covariance estimation errors
  • Works with few samples
  • Fixed sidelobes, no null steering
  • Limited angular resolution
  • Cannot resolve closely-spaced sources

MVDR (Capon)

  • Adaptive nulling — places nulls on interferers automatically
  • Higher angular resolution than DAS
  • Good for communications (SINR maximisation)
  • Requires matrix inversion — computationally heavier
  • Sensitive to steering vector errors
  • Needs sufficient data for covariance estimate
CHAPTER 09

GNU Radio and Python Workflow — From ADC to Beam

🛠
Hardware prerequisite for real-data DBF: You need multiple phase-coherent SDR receive channels. Two RTL-SDR dongles driven from the same clock (mod for shared oscillator) or a KrakenSDR (5-channel coherent RTL-SDR) are practical options. A HackRF or USRP with multiple RX boards also works.
1

Receive and synchronise I/Q streams

Each element's received I/Q samples arrive from its own SDR channel. The channels must be sample-synchronous (started at the same time) and phase-synchronous (same local oscillator). Use a calibration signal (known transmitter at known angle) to correct residual phase offsets between channels.

2

Buffer into data matrix X

Collect T time samples from all N elements into a matrix X of shape N × T. T should be large enough for reliable covariance estimation (T ≥ 10N) but small enough that the signal environment hasn't changed.

3

Compute covariance matrix R

R = X @ X.conj().T / T. Add diagonal loading: R += ε·I for numerical stability. ε is chosen as a fraction of the mean diagonal element of R.

4

Compute weight vector (DAS or MVDR)

Choose your beamformer. For a receive-only application tracking a known signal, DAS is fine. For interference suppression, use MVDR. Apply weights to X to get the output time series y.

5

Demodulate / display

The output y is a complex baseband signal — the same as any SDR output. Feed it to FM demodulation, FFT display, or further signal processing. You have effectively created a software-defined directional antenna.

GNU Radio provides blocks for multi-channel SDR input, matrix operations, and output. KrakenSDR ships with Python-based DBF and MUSIC DOA software designed exactly for this workflow.

CHAPTER 10

Practical Demo — Receive Two FM Stations, Steer a Null to Suppress One

This demonstration uses two phase-coherent SDR channels and a 2-element dipole array to show null steering in software. It requires two FM broadcast stations visible on your spectrum at different directions from the array.

🛠
Equipment: 2× RTL-SDR dongles with shared clock (or KrakenSDR), 2× dipoles at VHF FM spacing (λ/2 ≈ 1.4 m at 100 MHz), 1× laptop running Python 3 + NumPy + SciPy + PyRTLSDR or KrakenSDR Python SDK.

Procedure:

  1. Identify two FM stations on different frequencies (e.g., 98.4 MHz and 102.1 MHz) coming from clearly different directions (use CellMapper equivalent for FM broadcast, or simply rotate a portable radio).
  2. Tune both SDR channels to 100 MHz (wide enough to see both stations in the 2 MHz bandwidth).
  3. Collect 10,000 samples from both channels simultaneously. Form X (2 × 10000 matrix).
  4. Apply DAS steering toward station A's direction. Listen to the output — station A should be louder than station B.
  5. Compute MVDR weights with the steering vector for station A. Apply to X. The MVDR output should suppress station B (which now acts as an interferer) even more strongly than DAS.
  6. Swap: steer toward station B, use MVDR. Station A should now be suppressed.
  7. Listen to both outputs and confirm the spatial selectivity — you are hearing direction-based separation of two overlapping signals.

Knowledge Check

1. What is the steering vector for a 4-element ULA with λ/2 spacing, steered to θ = 0° (broadside)?

Show answer

a(0°) = [1, e^(j·0), e^(j·0), e^(j·0)] = [1, 1, 1, 1]. All elements have the same phase because sin(0°) = 0 — the signal arrives simultaneously at all elements when coming from broadside. The beamformer weights are the conjugate: w = [1, 1, 1, 1] / 4 (normalised).

2. Why does MVDR need the covariance matrix R, while DAS does not?

Show answer

DAS uses only the geometry (steering vector) to set weights, ignoring the actual interference environment. MVDR minimises the total output power subject to unity gain in the desired direction — to minimise power, it must know where the power is coming from, which is encoded in R. R captures the spatial distribution of interference and noise, enabling MVDR to place nulls precisely on strong interference sources.

3. A 2-element DBF system forms a null on an FM station at 98.4 MHz arriving from 45°. What MVDR weight vector achieves this, assuming unit gain in the broadside direction (θ = 0°)?

Show answer

This is a constrained optimisation — in practice you'd compute w_MVDR from the data. Conceptually: the weight for element 0 is 1 (reference), and the weight for element 1 introduces a phase that makes the 45° signal cancel while the 0° signal passes. For a 2-element system with λ/2 spacing, the null at 45° requires element 1 weight phase = −π sin(45°) × π = −π × 0.707 = −2.22 rad. Combined with the unity gain constraint, w = [1, e^(j·2.22)]^* / 2.

4. What does MUSIC provide that MVDR does not?

Show answer

MUSIC provides DOA (Direction of Arrival) estimation — it tells you where signals are coming from without needing to specify the target direction first. MVDR is a beamformer: you specify a direction and it maximises SINR from that direction. MUSIC uses the noise subspace of the covariance matrix to identify source directions with super-resolution capability (potentially resolving two sources closer than the Rayleigh limit).

5. Name two reasons why practical digital beamforming requires phase-coherent receivers across all elements.

Show answer

(1) Each element's I/Q samples must have a stable, known phase relationship to combine correctly. Independent oscillators drift independently, making the inter-element phase random and uncontrollable. (2) Steering vector calibration: to accurately map the steering vector to physical angles, you must know the phase offset between each receiver channel. Incoherent receivers would require continuous re-calibration even when the array geometry is fixed.

✅
Lesson 20 Summary: Software beamforming replaces hardware phase shifters with complex weight multiplication in code. The steering vector a(θ) = [1, e^(jkd sinθ), ...] describes how a signal from direction θ appears across array elements. Delay-and-sum applies the conjugate steering vector as weights — simple but with fixed sidelobes. MVDR (Capon) uses the received data covariance matrix to adaptively place nulls on interferers, outperforming DAS in interference-heavy environments. MUSIC provides super-resolution DOA estimation through eigendecomposition of the covariance matrix. All three algorithms are implementable in NumPy in under 20 lines of Python.
